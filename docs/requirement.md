把一个包含 .arw / .mp4 / .mov / .jpg 文件的文件夹（含子目录）整理到统一结构：

视频文件 → Video/<年份>/<YYYY-MM-DD>/文件名

照片文件 → Picture/<年份>/<YYYY-MM-DD>/文件名

基础功能需求拆解（MVP 版本）
1. 输入与扫描

输入源目录：指定一个文件夹，包含子目录。

递归扫描：遍历所有子文件夹，找到目标扩展名的文件。

照片类：.arw .jpg .jpeg 等等

视频类：.mp4 .mov 等等

2. 时间获取逻辑

优先级：

照片：从 EXIF 读取 DateTimeOriginal

视频：从 QuickTime/MP4 元数据读取 CreateDate 或 MediaCreateDate

如果没有元数据 → 使用文件的修改时间（mtime）

日期格式化：统一为 YYYY-MM-DD。

3. 分类逻辑

判断文件类型 → 照片归到 Picture，视频归到 Video。

根据拍摄日期生成目录层级：

/<DEST_ROOT>/Video/2024/2024-01-01/xxx.mov

/<DEST_ROOT>/Picture/2024/2024-01-01/xxx.jpg

4. 文件处理

拷贝模式（默认）：复制源文件到目标目录。

移动模式（可选）：复制成功后删除源文件。

冲突处理：如果目标文件夹已有同名文件，自动在文件名后加 _1、_2 …。

5. 最小输出

控制台输出日志：显示文件 → 目标路径的映射。

统计信息：最终输出处理的文件数量。

验收标准（MVP）

.arw/.jpg/.jpeg 文件被正确分类到 Picture/年/月日。

.mp4/.mov 文件被正确分类到 Video/年/月日。

子目录内的文件也能被扫描并整理。

元数据存在时按拍摄时间归类，不存在时按文件修改时间。

遇到重名文件不会覆盖，而是生成 _1/_2。

拷贝模式不删除源文件，移动模式会在复制成功后安全删除。
